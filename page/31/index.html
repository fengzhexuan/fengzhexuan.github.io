怎样使用CSS3伪类（一）

发表于 2011-05-23 | 分类于 随笔


　　又是新的一周了，这周开始一个全新的系列——怎样使用CSS3伪类。
　　伪类（pseudo class），对于大多数网页设计人员来说，是一个软肋。常常会对伪类产生各种各样的困惑，关于伪类，如果使用得当的话会达到事半功倍的效果。而在最新的CSS3标准中，列出了16种伪类，为我们提供了在新标准中强大的选择方法。
　　首先还是向大家介绍一下CSS中伪类的历史渊源吧。

伪类简史
　　CSS1标准完成于1996年，包括了一些直到今天还几乎天天使用的伪类。例如：
:link
:visited
:hover
:active
这些状态可以使用于任意元素，但是通常用于，从此以后便使用伪类这个叫法。你肯定难以想象，这些还早于HTML4标准的制定，而HTML4是在1997年的12月才发布。
　　CSS2降临
　　CSS2紧跟着CSS1的步伐，于两年后的1998年5月发布。随着令人期待的position一起出现的伪类有：
:first-child
:lang()

  • :lang
    　　有多种方法来表示文件的语言，在HTML5中，可以这样定义。当你的站点是随着不同的国家而动态变换语言的时候，现在你可以使用:lang(en)。
  • :first-child
    　　你可能已经使用过这个选择方法。通常用来添加或者移除一个列表中第一个元素的边框（border），然而在CSS2中却没有与此相对应的:last-child，直到CSS3中这两个兄弟才得以相见。
    为什么要使用伪类
    　　伪类能够使你的内容风格动态变化，这是使得如此有用的原因之一。在标签中，当用户与链接交互时可以有不同的状态显示。而新的伪类可以使文档内容基于其位置或者状态而动态变换形式。
    　　在     CSS Proposed Recommendation中,一共介绍了16种新的伪类，共分为四种：结构伪类，UI元素状态伪类，目标伪类，否定式伪类。

　　这就算是本系列的开篇吧，待我下次一一详细介绍这16个新伪类，敬请期待。

线框图——优秀的网页设计方式

发表于 2011-05-17 | 分类于 随笔


　　想必大家都知道“蝴蝶效应”，当一个小的事情经过时间与空间的放大之后会变得很难控制直至不可控。网页设计也是如此，在设计的最初如果有微小的差错，那么当所有的设计流程都结束后想要改正将会变得十分困难。
　　线框图即是这样一种优秀的设计方法，将差错扼杀在摇篮中，当然，你得运用得当。

什么是线框图？
　　线框图是一种最基本的视觉结构，展现了一个网页中多种元素的位置摆放。也可以说是辅助网站设计与开发的轮廓图。你可以使用各种工具来绘制线框图，甚至只使用一页白纸与一杆铅笔。我们最终的目的都是设计一个NB的网站。
为什么使用线框图？
　　作为一种工作模式
　　线框图帮助设计师着手一个网站设计，来决定需要什么元素，各个元素的摆放位置等。
　　帮助确定问题区域
　　在线框图的帮助下，能够容易地知道哪些区域会在后续的设计或开发过程中制造屏障。如果能在前期就能确定问题所在将是一件事半功倍的事情，因为在初期进行调整或者修正非常简单。
　　节省时间
　　时间就是金钱。线框图由于其简单的特性，使得其能够在短时间内建立并容易修改。另一方面，在网页设计的过程中修改会占用很大比例的时间，而线框图却能够提供一个坚实的基础。
　　与客户交流意见的最好形式
　　颜色和图形元素会转移客户的注意力，从而偏离了对网站本质问题的交流目的。而线框图却不包含这些内容，正是表现网页布局设计的最好形式。
谁来使用线框图
　　很明显，设计师需要使用线框图。开发人员也可以使用线框图来确定网站的功能技术细节。客户也适合使用，他们可以利用线框图在更早时候描述需求及其所要求的网站特性。

响应式设计介绍

发表于 2011-05-16 | 分类于 随笔


　　网络对于很多人来说已经是不可或缺的生活中的一部分。你可能有一台笔记本，一部手机，一个平板电脑，它们拥有不同的屏幕分辨率。现在屏幕的分辨率宽度，小到240px，大到2560px。用户不单单使用桌面电脑来浏览网页，所以传统的固定宽度设计已经不能满足需求了。这就必须使得网页能够随着屏幕的变化而适应变化。

　　可能你会说，可以使用fluid design。虽然它可以随着显示屏幕宽度的大小进行自适应，但是当这种设计在手机终端显示或者当宽度小于某个数值后，其显示效果就差强人意了。大家可以看这个Demo的效果，当然你需要调整浏览器的窗口大小。基于此，Responsive web design就应运而生了。

　　第一次关注Responsive web design是在农历年后不久，最初是看到A List Apart上面的这篇文章Responsive web design。看过之后，一直纠结这个Responsive web design应该如何翻译，目前自己感觉比较合适的是“响应式设计”。

　　响应式设计原理很简单，通过对不同的浏览器宽度用不同的CSS来进行渲染，从而达到自适应的目的。在宽度自适应中，只能给元素的宽度属性设置为比例，随着浏览器宽度的变化，只能让浏览器自己对其进行渲染，是不能够进行控制的。而响应式设计却能够控制在不同宽度下个元素的表现效果，这是两者最大的区别。

　　响应式设计需要浏览器对CSS3的支持，通过对不同宽度使用不同的CSS需要通过media query来实现，这也是该种设计方式还不能够大范围使用的主要原因。这里有一个响应式设计的Demo，来自Web Designer Wall。

　　下次向大家介绍响应式设计的具体实现方法，敬请期待~

2011草莓音乐节一日游

发表于 2011-05-12 | 分类于 随笔


　　四月份的时候在豆瓣上看到草莓音乐节第三天（五月二日）的阵容，心里便痒痒了，恰好碰上学校里文化节上在卖学生票。毫不犹豫地定下了第三天一定得去草莓音乐节逛逛。

　　这是第二次来看摩登天空公司的演出了，上一次是在零九年的十月份，看到了小河和痛仰。那么这次纯粹是奔着台湾舞台去的——胡德夫、黄玠、草莓救星，还碰上了后海大鲨鱼和周云蓬，真太爽啦。

　　赶集啦，文艺青年赶集了，在通州运河公园。这个偏远的公园其实真没什么文化，但是从通州北苑的地铁到公交车上的老司机，都知道这帮子年轻人是去看“草莓”滴。好像草莓是个羞答答的姑娘，等着人们去搭讪，她歌唱，等着人们来捧场。

　　这次草莓音乐节上有IMART创意市集，还有FAT ART的展览，后者我们去溜了一眼，里面有各种青年人和摇滚歌手的作品，其实在中国这个价值观比较趋同的社会，这些东西只能在一些偏僻的地方偏僻的时刻出现，而且并没有真正呈现出青年亚文化的蓬勃生机，从那些作品中很难捕捉到那种处于青年特有的黄金时期的生命力或者创造力，那些虚张的姿态的确是他们生命中的姿态，但仅仅只是个夸张的姿态而已。甚至会想到，几年以后，青年人慢慢走入中年，可能这些东西在他们的生活里会慢慢消失，这些乖戾的年轻人最终平和，连搞怪的心情都没有了。这是题外话。

　　这是我们看的第一个乐队——lazy camel，这个吉他手同学摘了眼镜跟街头的小青年一样。嘿嘿~

　　然后我们很快转战台湾舞台，舞台上空的帐篷布被风吹翻，让我们等了很久很久。然后，黄玠终于来了。先唱了些新歌，然后便是最爱的《香格里拉》，说实在的，他的歌跟郁达夫的小说一样表达着青年人的郁闷和彷徨，没有强大的行动力，只有矫情的小心绪。但是这样的小歌曲还是特别的可爱，在冷酷的社会里给尤其脆弱的爱好文艺的灵魂一点点慰藉。一边听着，一边关掉音乐鼓起足足的勇气。【黄玠推荐曲目 《25岁》《香格里拉》《我的高中同学》】

　　草莓舞台来了周云蓬大叔，刚开始这些歌曲不入左撇子同学的法耳，后来的《永隔一江水》留住了我，也留住了他，哈哈。他是个敢于面对生活面对现实的斗士，唱着一些像美国六十年代的那种抗议歌曲。他的吉他弹得真不错。【推荐歌曲：《一个儿童的共产主义梦想》】

　　甜梅号来啦，不过实在听不懂，冗长的后摇风格。

　　草莓救星的出现还是让我很兴奋，一个很可爱的乐队。【推荐曲目：《自在》《 sleep》】

　　胡德夫大叔最后出现，可惜只看了两曲。【推荐曲目 《匆匆》《牛背上的小孩》《太平洋的风》《最遥远的距离》】

　　后海大鲨鱼算是草莓的高潮，付菡手抓所有人的神经，山坡上的人们举起手，打着拍子，像过节一样地开心。

　　天黑了，草莓音乐节也结束了，集会散场。这些青年各回各家，却又找不到灵魂家园的归途。

最流行CSS3技术Top 10（五）

发表于 2011-05-12 | 分类于 随笔


　　接上次最流行CSS3技术Top 10（四），下面我们介绍Transform与Transition两种技术。

Transform（变形）
　　Transform可以做到让元素：旋转、比例、斜切、平移。这里我们以旋转为例进行说明，如果想全面了解请 谷歌之。它的参数如下：
transform: type(value) type(value) type(value) type(value);
　　对于旋转来说：
transform: rotate(value);
如果是正值的话，元素将会顺时针旋转，想法则会逆时针旋转。在IE中可以使用Matrix滤镜来实现相同的效果，但是不要忘记对使用该滤镜的元素添加zoom:1。

　　 这样一个完整的Transform声明就变成以下的样子：
.rotate {
-moz-transform: rotate(-5deg);
-o-transform: rotate(-5deg);
-webkit-transform: rotate(-5deg);
transform: rotate(-5deg);
-ms-filter: “progid:DXImageTransform.Microsoft.Matrix(M11=0.9961946980917455,M12=0.08715574274765817, M21=-sizingMethod=’auto expand’)”;
filter: progid:DXImageTransform.Microsoft.Matrix(sizingMethod=’auto expand’,M11=0.9961946980917455,M12=0.08715574274765817,M21=-0.08715574274765817,M22=0.9961946980917455);
zoom: 1;
}
Transition（变换）
　　该属性可以实现元素状态的平滑变换，它的参数如下：
transition:property duration timing-function delay;
其中，property代表需要进行变换的属性，duration表示变换持续的时间，timing-function表示计算时间的方法，delay表示变换延迟。
　　虽然现代浏览器都支持该特性，但渲染效果却有区别。相比Firefox来说，Chrome与Opera的变换效果更加平滑。在IE中的滤镜，有一些可以实现该效果，但是使用哪种，最好还是在微软官方网站上进行寻找。
　　 这样一个完整的Transition声明就变成以下的样子：

#photos > img:hover {
-moz-transform: scale(1.1,1.1);
-webkit-transform: scale(1.1,1.1);
-o-transform: scale(1.1,1.1);
transform: scale(1.1,1.1);
-moz-transition: all 1s ease-in-out;
-webkit-transition: all 1s ease-in-out;
-o-transition: all 1s ease-in-out;
transition: all 1s ease-in-out;
}
　　至此，最流行CSS3技术Top 10系列就向大家介绍完了，还请多多支持 尼龙扣。
