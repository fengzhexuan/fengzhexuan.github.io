关于7·23动车追尾事故的一些思考

发表于 2011-07-25 | 分类于 随笔

　　我一直以为自己不是个愤青，但当昨天刷了一天围脖后，我错了。虽然愤青，但也要做个理智的愤青。

　　7月23日晚上在温州的动车追尾事故牵动了亿万的国民，然而这次国民再也坐不住了。当一个人基本的衣食住行得不到保障的时候，这个社会就到了蜕变的时候了。

　　先来说说这场事故吧。以下是我综合网友以及个人的一些疑问。为什么会有那么多人走散？火车上的乘务人员有没有组织有效的疏散？有没有追尾的相应应急预案？车上到底有多少乘客？伤亡人数到底是多少？至于事故的原因，只能等待铁路部门来进行查明了，希望能给我们一个满意的答案。

　　事故已经发生，时光不能逆转。在为死伤者祈祷祝愿的同时，我们应该思考从中能总结出什么，学到什么。

　　事故的消息最先在新浪微博中传出，在事故发生后的短时间内迅速传播开来。而第二天的《人民日报》、《光明日报》、《经济日报》、《解放军报》这四大官媒报纸的头版头条却是《中央军委举行晋升上将军衔仪式》，这不得不让人痛心。在当今互联网通讯如此发达的年代，真相是掩盖不了的，群众的呼声是掩盖不了的。

　　消息在微博上的不胫而走、迅速传播是一把双刃剑，真相可以迅速传播，谣言也同样可以。微博上的有些消息很难辨认真假，只能凭自己进行判断。但这也反面映射出了，相关部门没能够及时地召开新闻发布会，导致了大家对事故原因的各种猜测以及各种疑问。昨天我就为还未将事故调查清楚就将损毁的车头进行填埋处理疑惑不解，官方先否认进行填埋，后来又在发布会上说是为了以后的处理工作做铺垫，这种理由太牵强，都埋了还怎么调查？

　　从事故发生到现在还没有明确乘客名单，不能打消群众心中的疑问，而死亡人数35人也让人心生怀疑。动车实名制后，没有打击到黄牛对车票的倒买倒卖却成为了验证死者身份的途径，这是何等的讽刺。建议升级火车的检票系统，可以借鉴地铁，在出入车站都进行刷卡，这样不仅可以对逃票加以打击，还能对发生事故后的乘车人员确定起到极大的作用。不知道现有的火车系统是否有这样的作用。

　　新闻中有这样一个细节——教观众如何使用安全锤，在事故已经发生后才来进行亡羊补牢式的普及。乘坐飞机时乘务员都会向大家讲解发生意外情况时应该如何反应，难道火车上就不用吗？这些基本的生存技能应该从娃娃抓起，而不只是大讲特讲各种大道理，几岁的小孩懂个屁啊。

　　前任铁道部长刘志军的案子还没有查明白，不知道高铁频发事故的原因到底是什么样的，倍感担心。

　　中国在经济高速发展的同时，也亟需一次蜕变，国民素质的提高、国家福利的保护、政治的改革，而我们这一代也将在这场巨大的蜕变中作出自己应作的努力。　　

　　人在做，天在看，不做亏心事，不怕鬼敲门。

【转载】24个必学的Google+小技巧（注册邀请）

发表于 2011-07-14 | 分类于 随笔


　　来源在这里。

　　1.阅读「首页」中的讯息时，按一下键盘的「j」可阅读下一则，按「k」阅读上一则，这跟Linux 里的vi 之类的编辑器很像。特别的是，按「j」 一则一则往下阅读翻到页尾时，会自动往下翻页，不必再用滑鼠按。

　　2.使用「j」、「k」按键浏览讯息时，可随时按「Enter」键发表回应。

　　3.如何使用粗体、斜体或在文字上面加删除线？
　　文字加粗： Vince Zhang（在文字前后加上星号「」，如：Vince Zhang* ）
　　文字倾斜： Vince Zhang（在文字前后加上底线「_」，如：Vince Zhang ）
　　文字加删除线： BRIIAN （在文字前后加上横线「-」，如：-BRIIAN- ）
　　注意！如果要使用中文字的删除线，请在「-」后面加个「.」，如「-不来恩-.」这样。

　　4.发文、回应或修改讯息时，可以按一下键盘上的「Tab」再按「Enter」键，即可快速送出。

　　5.发文时可选择接收对象与群组，直接按「新增分享对象」选取指定帐号或社交圈即可。只要你事先将不同类别的朋友安排在不同「社交圈」里，发文时就不用担心你跟好朋友的对话被其他不熟的人或父母、老师、同事或主管看到了。

　　6.发文或回覆讯息时，可以先输入「@」或「+」再紧接着输入使用者的名称（或Email帐号），此时会出现选单让我们点选使用者名称，即可直接在讯息或回应中显示对方的名字、指名回应。

　　7.发文后，发现打错字的话怎么办？在自己发出的讯息右上角的按钮按一下，可以选择编辑、删除、停用留言、停用转贴..等功能。

　　8.阅读讯息时，点一下上方的时间，可以直接开启该则讯息的专属网页，方便我们把网址分享给其他Google+ 站外的网友浏览。

　　9.阅读讯息时，点一下「限定开放」，可以查看阅读权限，看看有哪些人可以阅读该则讯息。

　　10.如果某则讯息很多人回应，一直占据「讯息串」的上方，让你很烦的话，可以在讯息右上角的箭头按一下，出现选单后再点「忽略这则讯息」，即可让他不再吵你。

　　11.Google+ 网页右上角的讯息通知区域除了可以显示全部的讯息通知总览，按上方的「较新」、「较旧」直接浏览通知的内容，不用点进去网页就可以知道最近的动态。除了用滑鼠按，也可以直接按键盘上的「←」、「→」左右键快速翻页。此功能支援Google 其他网站群，只要登入后有出现类似的通知工具列都可以直接在上面阅读你的Google+ 最新动态，不用开Google+ 网页也可以马上回应…。

　　12.在网页右上角的通知区域阅读最近的更新状态时，如果不希望某一则讯息一直跳出来吵你，可以按一下下方的「忽略这则讯息」让它安静点。

　　13.在「首页」点一下左边的「圈外讯息」，可以查看「他有加你、你没加他」的其他网友所分享的讯息。

　　14.在首页时按2下键盘的「q」，可快速跳到「即时通讯」的搜寻框，让我们选取要聊天的对象。除了即时文字一对一聊天之外，还可支援语音、视讯对谈、多人视讯会议。

　　15.在浏览讯息串里面的照片时，如果对方一次贴了2张以上的照片，按了缩图浏览时，可以用滑鼠滚轮或键盘「←」、「→」左右键快速翻页。看完后，按一下键盘的「ESC」按键即可跳回原本的地方。

　　16.如果你使用Google Chrome 浏览器，可以直接从桌面将图档拉到发言框，他会在背景自动上传图档，边上传可以边打字，不用再按一堆按钮选取、上传。

　　17.开启你自己上传的照片时，可以在照片右下角按一下「动作」选单，再按「编辑照片」，即可套用多个视觉特效。

　　18.发现网站有bug、功能不正常时，可以按一下网站右下角的「意见回馈」，直接用滑鼠圈选有问题的区块，然后再输入你要回报的问题即可。如果页面中有不希望被看到的隐私资讯，也可以先按「涂黑」按钮把隐私资讯涂黑再圈选回报。

　　19.当你启用Google+ 帐号后，可能会发现Email 信箱里面有一大堆通知信，如果要取消的话，请按一下右上角的设定图示，再点「Google+ 设定」，即可在设定页面中勾选要出现或不出现哪些通知信。

　　20.开启「这个」网页，登入后即可快速备份Picasa 相簿、个人资料、Google+ 讯息串的内容、Google Buzz 里面的内容与你的Gmail 通讯录与社交圈的等资料。

　　21.当我们编辑个人资料时，如果想确定这些关系到个人隐私的项目是否有藏好，可以点一下右上角的「用其他身分查看个人资料」，我们可以输入指定帐号看看对方是否能看到你要让他看的资讯，或者按「网路上所有的人」看看其他不认识的人会不会看到不该看的东西。从这边按一按就好，不用一直登出、登入。

　　22.编辑「个人资料」时，可以按一下左侧大头贴下方的社交圈图示，即可设定要不要显示或显示哪一群好友的图示。

　　23.如果朋友没帐号或懒得注册，可以先给他「这个」demo 版的页面试玩，里面有几个重点功能可以边按边展示。可惜目前只有英文版。

　　24.如果你有Android 手机，可以下载安装「Google+应用程式」，除了浏览、发讯息之外，还可支援打卡、拍照上传..等等功能，出门在外也能轻松Google+。

如何避免重新设计的失败

发表于 2011-07-04 | 分类于 随笔


　　重新设计（redesign）是一个设计师迟早会遇到挑战之一。对于现有的网站进行重新设计，很多人会想，这不是挺简单的事情么？我们或多或少都听过一些失败的重新设计的案例。不论网站的大小，对其进行重新设计充满了各种危险与陷阱，不得不说这是对一个设计师的挑战，而疏远已有的用户是重新设计中最大的潜在威胁。当然，网站越大，这种可能性越大。下面我们探讨一下重新设计中应该注意的问题。

正确的重新设计
　　当你打算对已有的网站或者Web应用进行重新设计时，除了技术方面还应该考虑一些其他方面的因素。要时刻牢记一点，重新设计后会不会让网站已有的常客迷路。在一开始就和你的用户交流
　　与设计新站点不同，要重新设计的网站已经有了一定的用户基础。在工作开始之前最好与他们交流，搞清楚他们是如何使用网站的，没准他们对你的网站还会有更加独到的见解。这样在重新设计时将他们考虑进来，从而不会破坏他们的使用习惯。对新老用户同时测试
　　通常来说，我们只会对新用户进行测试。当然，当你只重新设计网站的一小部分时可以只考虑新用户，但将整个网站重新设计时，必须对新老用户同时进行测试，听听用户的心声。保持新老网站一段时间
　　这一点对于应用型的网站极其重要。让用户自己选择合适切换至新版本能够减少他们的抱怨，同时可以让用户自由的在新旧之间切换。比如Twitter、Google Analytics，会同时提供新旧版本供用户使用。在变化后提供便利的反馈渠道
　　这一点不用多说。用户界面显著变化是提供引导使用教程
　　如果用户界面会有显著变化时，应该提供有效的引导教程，让用户迅速熟悉新界面。万一失败了怎么办？
　　万一重新设计失败了怎么办？无意诅咒大家，只是说万一。这时面对用户的各种抱怨，应该怎么办呢？危机公关
　　还记得前阵子会理县的PS事件化险为夷转而变为宣传广告么？如果重新设计后的网站引来用户大量的抱怨之声，第一件要做的事情就是对其作出及时的处理，让用户知道已经对他们的意见进行处理了，动用大量的渠道，譬如微博等平台，争取覆盖到所有的用户。不要害怕回滚
　　有时候，回滚是最佳的解决办法。这时候你才有事件对重新设计中已知的错误进行改正、对不好用的设计进行改进。这样才不至于流失大量的用户。
　　不论何时，对于网站数据的监控是非常重要的，在进行重新设计之后尤其重要。
