面向对象样式表（OOCSS）（二）

发表于 2012-02-16 | 分类于 随笔


　　在面向对象样式表（OOCSS）（一）中，介绍了面向对象样式表编写的两条基本思想，这次介绍其优势以及需要注意的地方。

OOCSS的优势
　　之前已经涉及到一些OOCSS优势的内容，现在将其总结于此。
更快的网站
　　这一优势是显而易见的，更小的CSS文件载入时间更短。读者可能已经注意到，OOCSS会使得HTML文件变大，但大多数情况下，CSS将会极大的弥补回来。
更易于维护
　　随着网站规模的变大，CSS的样式也跟着变多。每当你试着在CSS文件底部添加一条新的样式时，会担心这条样式会产生怎样的影响。采用OOCSS编写样式表时，新添的样式可在已有的样式基础上进行。易于维护的样式表使其本身更具鲁棒性，由于样式都是模块化的，当有新的人员进行工作是更易上手。
需要注意的地方
仍然可以使用id
　　如果采用OOCSS编写样式表，绝大多数的样式都是以class的形式存在的。这并不意味着不要使用id，而是尽量避免在选择器中使用id。
在小型站点中使用
　　对于小型的站点以及App来说，采用OOCSS会觉得大材小用，因为样式表通常都比较简单。所以，OOCSS尤其适用于大型站点。不过，如果想养成OOCSS的思维方式，最好从小处做起～
一些Tips
　　1.避免使用派生选择器（例如 .sidebar h3）；
　　2.避免使用id作为样式表现的切入点；
　　3.不要将class跟随在元素后（例如 div.header）；
　　相信在一段时间后读者会有更多自己的心得体会。OOCSS就介绍完鸟，敬请关注本博的更多内容～

面向对象样式表（OOCSS）（一）

发表于 2012-02-02 | 分类于 随笔


　　内容对于网站的重要性不言而喻，也即“内容为王”。从web开发者的角度来说，一些人认为“速度为王”。个人认为，需将这两者结合来不断的提升web体验。 目前的大环境下，前端工程师普遍重视Javascript的效率，但CSS却没有得到足够的重视。今天就先说说面向对象CSS（OOCSS），它不仅可以提升网页的表现效率且可提高稳定性。

OOCSS的准则
　　像其他面向对象的高级语言一样，OOCSS的目的是鼓励代码重用，甚而编写出方便增减和维护的样式表。正如 这里所述，OOCSS主要基于两条准则。
　　将结构与表现分离
　　几乎每个网页上的元素都具有不同的视觉特征（即表现），在不同的上下文中反复。想想一个网站的品牌，颜色、微妙的渐变色、或者可见的边框。另一方面，一般其他无形的特点（即结构）也会反复。当这些不同的特点抽象成以类为基础的模块后，它们便成为可重用的代码。让我们对比两组简单的CSS代码进一步理解这条准则。

#button {
width: 200px;
height: 50px;
padding: 10px;
border: solid 1px #ccc;
background: linear-gradient(#ccc, #222);
box-shadow: rgba(0, 0, 0, .5) 2px 2px 5px;
}

#box {
width: 400px; overflow: hidden;
border: solid 1px #ccc;
background: linear-gradient(#ccc, #222);
box-shadow: rgba(0, 0, 0, .5) 2px 2px 5px;
}

#widget {
width: 500px;
min-height: 200px;
overflow: auto;
border: solid 1px #ccc;
background: linear-gradient(#ccc, #222);
box-shadow: rgba(0, 0, 0, .5) 2px 2px 5px;
}
　　以上代码典型的将结构与表现混为一起，经过观察将结构与表现分离之后可以得到以下的代码。

.button {
width: 200px; height: 50px;
}

.box {
width: 400px;
overflow: hidden;
}

.widget {
width: 500px;
min-height: 200px;
overflow: auto;
}

.skin {
border: solid 1px #ccc;
background: linear-gradient(#ccc, #222);
box-shadow: rgba(0, 0, 0, .5) 2px 2px 5px;
}
　　以上的代码将视觉表现提炼为skin类，从而达到了重用的目的。不仅代码量减少了，且利于修改。
　　将容器与内容分离
　　第二条准则是将容器与内容分离，为了说明这条准则的重要性，我们先看一下的CSS代码。

#sidebar h3 {
font-family: Arial, Helvetica, sans-serif;
font-size: .8em;
line-height: 1;
color: #777;
text-shadow: rgba(0, 0, 0, .3) 3px 3px 6px;
}
　　该样式将会应用在 #sidebar 子元素的所有 h3 中。同时，如果你想在 #footer 中的 h3 元素与 #sidebar 中的有相同效果，但是字体大小与 text-shadow 不同，则代码将会变成：

#sidebar h3, #footer h3 {
font-family: Arial, Helvetica, sans-serif;
font-size: 2em;
line-height: 1;
color: #777;
text-shadow: rgba(0, 0, 0, .3) 3px 3px 6px;
}

#footer h3 {
font-size: 1.5em;
text-shadow: rgba(0, 0, 0, .3) 2px 2px 4px;
}
　　或者会有更差的结果：

#sidebar h3 {
font-family: Arial, Helvetica, sans-serif;
font-size: 2em;
line-height: 1;
color: #777;
text-shadow: rgba(0, 0, 0, .3) 3px 3px 6px;

#footer h3 {
font-family: Arial, Helvetica, sans-serif;
font-size: 1.5em;
line-height: 1;
color: #777;
text-shadow: rgba(0, 0, 0, .3) 2px 2px 4px;
}
　　显然，我们复制了多余的样式，可能没有意识到这点（或者只是不将其放在心上）。OOCSS中，我们鼓励在编写代码前进行更多的思考，将各元素中通用的特点提炼出来（此处可参考面向对象编程）。通过继承选择器实现的样式是不可重用的（比如上例中的 #sidebar 与 #footer ）。采用OOCSS编程要注意将样式与容器元素分离，这意味着它们将可被重用而不用担心上下文结构。
　　本文先介绍OOCSS的两条基本准则，还有更多后续内容，敬请关注～

2011年总结（顺祝新春愉快）

发表于 2012-01-22 | 分类于 随笔


　　总结这个东西以前是被别人逼着写，现在竟然有自己想写的冲动，看来觉悟提高了一点。
　　2011年关于个人的目标算是基本完成：
　　1. 搭起了博客尼龙扣；
　　2. 选择了一份纠结的死去活来的工作，希望这个选择能够带来好运；
　　3. 坑爹的论文总算先把坑挖好了。
　　这一年，忙里偷闲跟右撇子童鞋去了不少地方、吃了不少东西，完成了一次旅行（虽然不远），在向吃货靠拢的道路上渐行渐远。
　　这一年，深受右撇子童鞋的影响，很少看除了课本之外书籍（简称课外书）的我看了一些书，足以证明以前的懒惰和幼稚。书中虽然有很多的道理，但还是要自己琢磨体会。
　　这一年，希望永远年轻的父母变老许多，以前对他们的关心实在太少，是到转变角色的时候了，从“被爱”到“爱”。
　　这一年，看到鲜活的生命殒灭，自叹生命是如此脆弱的同时，至今仍不想相信这是事实。不论遇到何种困难都要想想肩负的责任，调整好心态。希望TA能在天堂安好。
　　这一年，收获了一些，懂得了一些。
　　感谢父母、右撇子童鞋、家人、同学、朋友，所有这一年帮助过我的人。
　　在除夕前写下这些，祝愿家人与朋友身体健康、万事顺利！祝愿云游至此的童鞋龙年大吉！

前端工程师的必备技能

《地球上的星星》观后

发表于 2012-01-08 | 分类于 随笔


　　《地球上的星星》是《三傻大闹宝莱坞》中兰乔的扮演者阿米尔·汗自导自演的印度电影，这两部电影同样关注教育问题，不同的是一个小学、一个大学。

　　这部长达将近3小时的电影，以一个寄宿学校中新美术老师 Ram Shankar Nikumbh 的出现自然的分为上下两部分。上半部分描述了有读写障碍的小朋友 Ishaan Awasthi 是怎样被家庭作业、考试、分数折磨的死去活来，下半部分描述了一个美术老师如何将该小朋友带出沼泽的过程。全剧在一场美术比赛达到高潮，相信你看到这里便会露出会心的微笑抑或感动的泪水。当然该片不会缺少载歌载舞的印度电影特色，暂且不表。

　　这本电影让我想起来上幼儿园的情景，下面这个故事我一直将其作为自嘲式的笑话讲的。记得上幼儿园的第一天，老师在黑板上写了个“人”字，让小朋友们模仿，作为左撇子的我自然是用左手握笔，高兴的在小本子上写了一排“入”字。现在还对已被拆迁的老房子墙上反写的“马、牛、羊”有依稀的印象。

　　每个来到世界上的小朋友都是具有某种天赋的，不过这种天赋被具有特殊国情的教育无情的摧残了。

　　你我曾经都是特别的小星星，那现在呢，我想你心中会有答案了。
