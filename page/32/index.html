最流行CSS3技术Top 10（四）

发表于 2011-05-09 | 分类于 随笔


　　接上次最流行CSS3技术Top 10（三），下面我们介绍Gradient（渐变）与Multi-Background（多重背景）两种技术。

Gradient（渐变）
　　Gradient同样是CSS3中优美特性之一。在CSS3之前，通常实现渐变效果需要背景图片，但在CSS3中让浏览器来渲染渐变的效果，可以节省文件大小，节省带宽。虽然它有如此多的好处，但是需要付出代价，Gradient是一个具有众多参数属性的特性。
　　目前只有Firefox与Chrome支持该特性，虽然IE有实现线性渐变的滤镜，但却不能实现角度渐变。

　　 这样一个完整的Gradient声明就变成以下的样子：
.gradient {
color: #fff;
background: #aaaaaa url(gradient_slice.jpg) 0 0 x-repeat; /在不支持该特性的浏览器中依然使用背景图片/
background-image: -moz-linear-gradient(top, #07407c, #aaaaaa);
background-image: -webkit-gradient(linear,left top,left bottom,color-stop(0, #07407c),color-stop (1, #aaaaaa));
-ms-filter: “progid:DXImageTransform.Microsoft.gradient(startColorStr=’#07407c’,EndColorStr=’#aaaaaa’)”;
filter:progid:DXImageTransform.Microsoft.gradient(startColorStr=’#07407c’, EndColorStr=’#aaaaaa’);
}

　　Multi-Background（多重背景）
　　在CSS3中，你可以同时使用多个背景。该特性的参数非常简单，只需要用逗号将不同的图片分隔开来。但需要注意的一点是，只有在最后的图片中可以设定背景颜色。它的参数如下：
background:
image position size repeat attachment box,
image position size repeat attachment box,
image position size repeat attachment box,
image position size repeat attachment box color;

　　最新的浏览器都支持该特性，IE9还同时支持多重背景颜色。如果想要在所有浏览器中实现相同的效果，你需要在不支持该特性的浏览器中使用合成后的背景图片。
　　 这样一个完整的Multi-Background声明就变成以下的样子：

#wrap {
background: url(body-full.gif) top left no-repeat;
background:
url(body-top.gif) top left no-repeat,
url(body-bottom.gif) bottom left no-repeat,
url(body-middle.gif) left repeat-y;
-ms-filter: “progid:DXImageTransform.Microsoft.AlphaImageLoader(src=’ body-top.gif’,sizingMethod=’scale’)”;
filter: progid:DXImageTransform.Microsoft.AlphaImageLoader(src=’body-top.gif’, sizingMethod=’scale’);
zoom: 1;
}

　　当然你也可以这样写：
body {
background: url(all_layers.jpg) no-repeat 0 0;
background-image:url(first_layer.png),url(second_layer.png),url(third_layer.png),url(fourth_layer.png);
background-repeat: no-repeat, no-repeat,no-repeat, no-repeat;
background-position: 0 0, 0 50, 0 100, 0 200;
-ms-filter: “progid:DXImageTransform.Microsoft.AlphaImageLoader(src=’ body-top.gif’,sizingMethod=’scale’)”;
filter: progid:DXImageTransform.Microsoft.AlphaImageLoader(src=’body-top.gif’, sizingMethod=’scale’);
zoom: 1;
}

　　关于多重背景更好玩的应用可以去阅读 有趣的CSS3多重背景。
　　下次也就是该系列的最后一讲了，将讲解Transform与Transition这两种有趣的属性，敬请关注！

最流行CSS3技术Top 10（三）

发表于 2011-05-03 | 分类于 随笔


　　接上次最流行CSS3技术Top 10（二），下面我们介绍Box-shadow与Text-shadow两种技术。

Box-shadow
　　Box-shadow是我们期待已久的特性，该特性使某个元素具有阴影的效果。Box-shadow的参数如下：
box-shadow: position x-offset, y-offset, blur radius;
　　position：的默认值为outset，它的另外一个值为inset。
　　x-offset：正值表示阴影在元素的右面，负值表示在左面。
　　y-offset：正值表示阴影在元素的下面，负值表示在上面。
　　blur radius：该值如果为零，则阴影没有模糊很锐利。
　　 其实，如果大家对Photoshop有一定了解的话，Box-shadow与图层的外阴影的设定几乎一致。
　　现代浏览器全部支持该属性，但4.0以下的Firefox仍然需要添加前缀-moz，在IE9以下的浏览器中可以使用滤镜实现阴影效果。

　　 这样一个完整的Box-shadow声明就变成以下的样子：
.boxshadow {
-moz-box-shadow: 3px 3px 10px #333;
-webkit-box-shadow: 3px 3px 10px #333;
box-shadow: 3px 3px 10px #333;
/ For IE 8 /
-ms-filter: “progid:DXImageTransform.Microsoft.Shadow(Strength=4, Direction=135,Color=’#333333’)”;
/ For IE 5.5 - 7 /
filter: progid:DXImageTransform.Microsoft.Shadow(Strength=4, Direction=135,Color=’#333333’);
}
　　Text-shadow
　　Text-shadow与Box-shadwo是近亲，不过Text-shadow是为文字添加阴影的效果。利用该效果可以使文字生成外发光、浮雕、凸版印刷的外观效果。该特性的参数与Box-shadow基本相同，只不过position没有inset值。现代浏览器全部支持该属性，在IE9以下的浏览器中可以使用滤镜实现该效果。
　　 这样一个完整的Text-shadow声明就变成以下的样子：
.textshadow h2 {
text-shadow:1px 1px 2px rgba(48,80,82,0.8);
-ms-filter: “progid:DXImageTransform.Microsoft.Shadow(Strength=2,Direction=135, Color=’#305052’)”;
/ For IE 5.5 - 7 /
filter: progid:DXImageTransform.Microsoft.Shadow(Strength=2, Direction=135,Color=’#305052’);
}
　　下次将讲解Gradient（渐变）与多重背景这两种属性，敬请关注！

最流行CSS3技术Top 10（二）

发表于 2011-05-01 | 分类于 随笔


　　接上次最流行CSS3技术Top 10（一），下面我们介绍RGBA与Border-Radius两种技术。

RGBA
　　RGBA是CSS中众多定义颜色方法的一种，唯一不同的是在定义颜色的同时可以设置透明度。RGBA的参数如下：
property: rgba(r color value, g color value, b color value, opacity value);
　　其中color value的值从0至255，opacity value的值从0.0至1.0。现代所有浏览器都支持该属性，包括IE9。但是在IE9以下的浏览器中，需要使用滤镜来实现透明的效果。

　　 这样一个完整的RGBA声明就变成以下的样子：
.rgba {
background-color: #ff0000;
background-color: transparent; / 在IE8中需要使用transparent以使滤镜生效 /
background-color: rgba(255, 0, 0, 0.3);
-ms-filter: “progid:DXImageTransform.Microsoft.gradient(startColorstr=#4CFF0000,endColorstr=#4CFF0000)”;
/ IE8滤镜 /
filter: progid:DXImageTransform.Microsoft.gradient(startColorstr=#4CFF0000,endColorstr=#4CFF0000);
/ IE8以下的滤镜 /
}
　　Border-radius（圆角边框）
　　Border-radius能够实现圆角边框的效果，可以说它是CSS3中最有用的属性之一，让四方的边框变得生动好看起来。当四角的值都相同时可以使用简写。Border-radius的参数如下：
border-radius: top value, right value, bottom value, left value;
　　关于Border-radius在不同浏览器中实现使用不同的前缀，但是在IE9以下的浏览器没有滤镜能够实现相同的效果。最简单的方法是对Border于Border-radius都进行设置，在老旧的浏览器只实现四方的边框。
　　 这样一个完整的Border-radius声明就变成以下的样子：
　　当四角的值相同时
.roundcorners {
-moz-border-radius: 20px;
-webkit-border-radius: 20px;
border-radius: 20px;
}
　　当四角的值不同时
.roundcorners {
border: 5px solid #aaa;
-moz-border-radius: 10px 5px;
-webkit-border-top-left-radius: 10px;
-webkit-border-top-right-radius: 5px;
-webkit-border-bottom-right-radius: 10px;
-webkit-border-bottom-left-radius: 5px;
border-radius: 10px 5px;
}
　　下次将讲解Box-shadow与Text-shadow这两种属性，敬请关注！

谁的嗓门高

发表于 2011-04-30 | 分类于 随笔


　　“58同城~”随着这一嘹亮高亢的喊声，我把头扭向正在播放广告的移动电视屏幕。原来是58同城的电视广告。这不得不让人想起来过年后不久融得巨资且在各种媒体高调打出广告的“赶集网”，正是因为姚晨赶着驴去赶集的这支广告，还引起了一场“赶驴风波”，百姓网第一时间抢注了“赶驴网”。虽然在愚人节那天百姓网在其博客上表示，正式撤下“赶驴网”，并将赶驴网域名解析重定向至百姓网。

　　不到一个月，58同城也推出了电视广告，而且跟赶集网的广告如出一辙，丝毫没有创意可言。代言人杨幂在最后用双手摆出一个5，一个8，并指向58同城的Logo，老土的造型。该支广告甚至没有赶集网的制作精良，虽然让大家误解为赶驴，但那头呆呆的毛驴还是深入人心的。

　　愚以为，一个优秀的互联网产品，拼的不是嗓门，而是服务。信息聚合网站更是如此，用户如何能够在短时间内找到自己所需要的信息，而这些信息又是否真实。只有不断改进的服务，舒服的用户体验才能让用户感到满意，这才是检验标准，而且是唯一的。

　　在不长的乘车时间里，“58同城”和“赶集网”的嚎叫广告轮番上阵，摧残我们脆弱的神经以至于反感恶心。余音绕梁，N日不绝。如果商家想要达到这种效果的话，那他们的确做到了，而且做得很好。

最流行CSS3技术Top 10（一）

发表于 2011-04-29 | 分类于 随笔


　　时下HTML5与CSS3持续火热，不仅最新的浏览器对其都有了不同程度的支持。这些最新技术的应用不仅可以实现各种特殊效果，而且可以节省带宽。这里列出了CSS3中最流行的10种技术。

1. @font-face（服务器端字体）
　　@font-face允许用户从服务器上下载站点所用字体到个人计算机上，从而正确对页面进行渲染。其格式如下：
@rule-name {
font-family: “Fontname”;
src: url(fontfilename.ext) format(‘formatname’);
}

　　@font-face是唯一一个目前所有浏览器都支持的CSS3特性，不过IE9一下的浏览器只支持.eof格式的字体，IE9增加了对.woff格式的支持。不过有两个bug需要注意。在IE9以下的浏览器中，即使正确的声明了@font-face属性，还是不能够正确的渲染字体，要解决这个问题，需要对@font-face进行两次声明，第一次是.eof自己，然后按照@font-face的标准格式声明。在WebKit内核浏览器中同样有个bug，当对@font-face中声明的字体应用font-weight: bold 或者 font-style: italic 时不会起作用，解决方法是将font-weigh，font-style:normal，font-variant属性同时设定为normal以确定一个基准。
　　 这样一个完整的@font-face声明就变成以下的样子：
@font-face {
font-family: ‘Colaborate Light’;
src: url(‘ColabLig.eot’);
src: local(‘☺’), url(‘ColabLig.woff’) format(‘woff’),
url(‘ColabLig.ttf’) format(‘truetype’);
font-weight:normal;
font-style:normal;
font-variant:normal;
}
　　2. Opacity（透明度）
　　顾名思义，你可以对一个元素的透明度进行控制，从完全透明到不透明。透明度的属性会被子元素继承，如果想要达到更加精细的控制效果，则需要使用RGBA属性，在下一小节中我们会介绍到。
opacity: ;
　　其中，value的范围从0.0到1.0。但令人失望的一点是IE浏览器完全不支持Opacity属性，但微软特别提供了一个滤镜效果来实现透明度的改变。较短的写法如下所示：
filter: alpha(opacity=40);
下面是扩展的写法：
filter: progid:DXImageTransform.Microsoft.
Alpha(opacity=40);

使用IE的前缀写法如下：
-ms-filter: “progid:DXImageTransform.
Microsoft.Alpha(opacity=40)”;

　　 这样一个完整的Opacity声明就变成以下的样子：
.opacity {
background-color: #3C4C55;
color: #fff;
opacity: 0.5;
-ms-filter: “progid:DXImageTransform.Microsoft.
Alpha(opacity=20)”;
/ 上面一行只在IE8中有效 /
filter: progid:DXImageTransform.Microsoft.
Alpha(opacity=20);
/ 上面这一行对IE9以下均有效 /
filter: alpha(opacity=20);
}

　　下次将讲解RGBA与Border-radius这两种属性，敬请关注！

shot with camera or gun——《放大》观后

发表于 2011-04-26 | 分类于 随笔

　　这个片属于比较沉闷的电影，但是有着很深的哲学思考。

　　一位以拍摄时尚照片为生的摄影家，外出至公园拍照片，一对情侣引起了他的注意。他迅速按下快门，没想到被那个镜头里的女人发现。女人穷追猛打找摄影师要回照片，摄影师把女人终于打发走了，便开始洋洋得意地欣赏自己为这对情侣拍摄的照片。没想到这些平静的照片居然露出一宗凶杀案的蛛丝马迹。这位摄影师便开始执着地寻找这起谋杀案的真相，结果他自己也辨不清虚实了。

　　这个片子末尾的一节比较有趣，摄影师因为没有找到真相感到特别沮丧，在回家的路上看到一群游行示威的戏剧系学生在网球场里徒手表演着一场网球比赛，网球（其实什么都没有，只是演员的目光沿着“球”的飞行划出了轨迹）飞出了铁丝网，众人把目光投向摄影师，他只好服从众人意愿也加入了这场荒诞的虚拟比赛。在空地上表演捡球的动作，并把球扔了进去——其实他手中空空。个人的清醒在大众的狂潮里被淹没，现代艺术是不是把艺术家的无聊当做有趣？

　　拍摄了一张照片，摄影师貌似得到了真相，但是真相远远还没有被他掌握——看到并不意味着理解。这就是影像的诡谲之处。摄影式的观看虽然貌似客观和真实，但观看者远远无法触及时间和空间的某个鲜活的断面。

　　现在端着相机到处拍照的人越来越多，整个世界好像都成了照相机的猎物。只是鸟枪换成了相机，取景框内的风景在咔嚓之后成为了流逝岁月的一个切片，没有了生命力，成了属于过去的不会说话的证据。
